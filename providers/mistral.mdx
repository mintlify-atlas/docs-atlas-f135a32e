Mistral AI provides powerful language models through an OpenAI-compatible API. docker-agent includes built-in support for Mistral as an alias provider.

## Setup

1. Get an API key from [console.mistral.ai](https://console.mistral.ai/).
2. Set the environment variable:

```bash
export MISTRAL_API_KEY="your-api-key"
```

## Configuration

<Tabs>
  <Tab title="Inline">
    ```yaml
    agents:
      root:
        model: mistral/mistral-large-latest
        description: Assistant using Mistral
        instruction: You are a helpful assistant.
    ```
  </Tab>
  <Tab title="Named model">
    ```yaml
    models:
      mistral:
        provider: mistral
        model: mistral-large-latest
        temperature: 0.7
        max_tokens: 8192

    agents:
      root:
        model: mistral
        description: Assistant using Mistral
        instruction: You are a helpful assistant.
    ```
  </Tab>
</Tabs>

## Available models

| Model | Description | Context |
| --- | --- | --- |
| `mistral-large-latest` | Most capable Mistral model | 128K |
| `mistral-medium-latest` | Balanced performance and cost | 128K |
| `mistral-small-latest` | Fast and cost-effective | 128K |
| `codestral-latest` | Optimized for code generation | 32K |
| `open-mistral-nemo` | Open-weight model | 128K |
| `ministral-8b-latest` | Compact 8B parameter model | 128K |
| `ministral-3b-latest` | Smallest Mistral model | 128K |

For the latest model list, see the [Mistral documentation](https://docs.mistral.ai/getting-started/models/).

## Thinking budget

Mistral models support thinking mode with effort level strings. The default effort is `medium`.

```yaml
models:
  mistral:
    provider: mistral
    model: mistral-large-latest
    thinking_budget: high  # minimal | low | medium | high | none
```

To disable thinking:

```yaml
models:
  mistral:
    provider: mistral
    model: mistral-large-latest
    thinking_budget: none
```

## Auto-detection

When running `docker agent run` without a config file, docker-agent automatically detects available providers. If `MISTRAL_API_KEY` is set and higher-priority providers (OpenAI, Anthropic, Google) are not configured, Mistral is selected with `mistral-small-latest` as the default model.

## How it works

Mistral is a built-in alias provider:

- **API type:** `openai_chatcompletions`
- **Base URL:** `https://api.mistral.ai/v1`
- **Token variable:** `MISTRAL_API_KEY`

## Example: code assistant

```yaml
agents:
  coder:
    model: mistral/codestral-latest
    description: Expert code assistant
    instruction: |
      You are an expert programmer using Codestral.
      Write clean, efficient, well-documented code.
      Explain your reasoning when helpful.
    toolsets:
      - type: filesystem
      - type: shell
      - type: think
```
